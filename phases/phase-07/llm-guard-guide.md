# LLM Guard - AI Security Platform Guide

## Overview

LLM Guard is a security toolkit that protects Large Language Model (LLM) applications from adversarial attacks, data leakage, and inappropriate outputs. It's a critical component of the AI Security Platform, addressing OWASP LLM Top 10 vulnerabilities.

| Aspect | Details |
|--------|---------|
| **Project** | [protectai/llm-guard](https://github.com/protectai/llm-guard) |
| **License** | MIT |
| **Approach** | ML-based scanners (HuggingFace models) |
| **Deployment** | Kubernetes (FastAPI service) |

## HuggingFace Models Explained

### Why ML Models Instead of Regex?

| Approach | Pros | Cons | Example |
|----------|------|------|---------|
| **Regex** | Fast, simple, predictable | Easy to bypass | `/ignore.*instructions/` |
| **ML Models** | Understands context & intent | Slower, larger | DeBERTa transformer |

**Example Attack:**
```
Regex pattern: "ignore.*instructions"
  âœ… Blocks: "Ignore all instructions"
  âŒ Misses: "Disregard everything above"
  âŒ Misses: "Pretend the previous context doesn't exist"

ML Model (DeBERTa):
  âœ… Blocks: "Ignore all instructions"
  âœ… Blocks: "Disregard everything above"
  âœ… Blocks: "Pretend the previous context doesn't exist"
  âœ… Blocks: Obfuscated attacks, Unicode tricks, etc.
```

### Why CPU-Only PyTorch?

**Problem Encountered:**
```
pip install llm-guard
  â†’ Downloads torch with CUDA (~2GB)
  â†’ Downloads nvidia-cudnn (~700MB)
  â†’ OOM killed (exit code 137)
```

**Solution:**
```bash
# Install CPU-only PyTorch FIRST
pip install torch --index-url https://download.pytorch.org/whl/cpu

# Then install llm-guard (won't download CUDA libs)
pip install llm-guard
```

**Result:**
- PyTorch CPU: ~200MB (vs ~2GB with CUDA)
- No NVIDIA libraries needed
- Works fine for inference (slower but sufficient)

## Lessons Learned

| Issue | Cause | Solution |
|-------|-------|----------|
| OOM during pip install | PyTorch downloads CUDA libs | Use `--index-url .../cpu` |
| Startup probe timeout | Model downloads take 5+ min | Increase timeout to 20min |
| Slow first request | Lazy model loading | Call `/warmup` on startup |
| High memory usage | All models in RAM | Could use ONNX for optimization |
